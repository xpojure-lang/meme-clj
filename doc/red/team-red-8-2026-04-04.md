# Red Team 8 Assessment Report

**Date:** 2026-04-04
**Branch:** `main` @ `45b8d2a`
**Methodology:** 7 parallel adversarial agents, 62 hypotheses, REPL-verified
**Tools used:** clojure-mcp (nREPL), clojure-lsp, clj-kondo, bb CLI, web research

## Executive Summary

62 hypotheses tested across 7 probe areas (tokenizer, parser, formatter, value resolution, static analysis, CLI, cross-platform). **19 confirmed findings**: 1 P0, 3 P1, 10 P2, 5 P3. The most critical issue is that `#?@()` reader conditional splicing is completely non-functional in eval mode (P0). Three P1 issues affect auto-resolve namespaced maps and reader conditional nil injection.

## Findings by Severity

### P0 -- Critical

**F1. Reader conditional splice `#?@()` is non-functional in eval mode**
- Hypothesis: H18-ext
- Input: `[0 #?@(:clj [1 2]) 3]` on JVM
- Expected: `[0 1 2 3]` (Clojure behavior)
- Actual: `[0 [1 2] 3]` (nested vector, not spliced)
- Root cause: `cst_reader.cljc:295` sets `:meme/splice` metadata on the matched branch, but no code in the codebase reads this metadata to perform the splice. The marker is created and ignored.
- Impact: Any `.meme` code using `#?@()` produces silently wrong results.

### P1 -- High

**F2. Reader conditional injects phantom `nil` when no platform matches**
- Hypothesis: H18
- Input: `[1 #?(:cljs 2) 3]` on JVM
- Expected: `[1 3]` (Clojure behavior -- unmatched branch produces no value)
- Actual: `[1 nil 3]`
- Root cause: `cst_reader.cljc:291-297` -- `some` returns `nil` when no platform matches, and this `nil` is included in the form vector. No sentinel distinguishes "matched nil" from "no match".
- Impact: Devastating in maps: `{:a 1 #?(:cljs :b) #?(:cljs 2) :c 3}` produces `{:a 1, nil nil, :c 3}`.

**F3. `#::{}` auto-resolve namespaced map produces corrupt keys**
- Hypothesis: H62a
- Input: `#::{:a 1}`
- Expected: Deferred auto-resolve keywords (like standalone `::a`)
- Actual: `{:/a 1}` -- keys get empty-string namespace via `(keyword "" "a")`
- Root cause: `cst_reader.cljc:263` -- `(subs "#::" 3)` yields `""`, passed to `(keyword "" k-name)`.
- Impact: Invalid keywords that Clojure's reader rejects.

**F4. `#::{}` roundtrip loses a colon**
- Hypothesis: H62b
- Input: `#::{:a 1}`
- Roundtrip output: `#:{:a 1}` (one colon, not two)
- Root cause: `printer.cljc:343` -- `(str "#:" ns-str "{")` with `ns-str = ""` produces `#:{}`. No flag distinguishes `#::` from `#:`.
- Impact: Source notation destroyed on roundtrip.

### P2 -- Medium

**F5. Duplicate map keys silently accepted**
- Hypothesis: H12
- Input: `{:a 1 :a 2}` -- meme produces `{:a 2}` (silent last-wins). Clojure throws "Duplicate key".
- Location: `cst_reader.cljc:168` -- `(apply array-map items)`

**F6. Duplicate set values silently accepted**
- Hypothesis: H13
- Input: `#{1 1}` -- meme produces `#{1}`. Clojure throws "Duplicate key".
- Location: `cst_reader.cljc:175` -- `(set items)`

**F7. `#::alias{}` loses alias distinction in roundtrip**
- Hypothesis: B1
- Input: `#::str{:a 1}` roundtrips to `#:str{:a 1}` (alias → explicit). The CST reader stores `ns-str = "str"` for both `#:str` and `#::str`.

**F8. `#:{:a 1}` (bare `#:` with no namespace) accepted**
- Hypothesis: H62c
- Clojure rejects this as "Namespaced map must specify a valid namespace: null". Meme silently produces corrupt `:/a` keys.

**F9. Ratio with signed denominator accepted**
- Hypothesis: Extra-A
- Input: `1/-2` produces `-1/2` in meme; Clojure throws "Invalid number".
- Location: `resolve.cljc:213-224` -- `BigInteger` constructor accepts signed strings.
- Fix: Validate denominator doesn't start with `+` or `-`.

**F10. Octal string escapes more permissive than Clojure**
- Hypothesis: Extra-B
- Input: `"\0a"` produces `[NUL, 'a']` in meme; Clojure throws "Invalid digit".
- Location: `resolve.cljc:64-85` -- greedy 1-3 digit consumption without terminator check.

**F11. Invalid auto-resolve keywords accepted at read time**
- Hypothesis: H41
- `::`, `::/`, `::/foo`, `::foo/` all pass through as deferred keywords. Clojure rejects all as "Invalid token".
- Impact: Propagate invalid tokens through `meme->clj` and formatter.

**F12. Unpaired surrogates accepted as symbol characters**
- Hypothesis: H2
- `\uD800` passes `invisible-char?` and is valid in symbols. Survives in-memory roundtrip but is a time bomb for any UTF-8 I/O.
- Location: `tokenizer.cljc:59-66` -- missing range `(<= 0xD800 c 0xDFFF)`.

**F13. U+2007 FIGURE SPACE: cross-platform divergence**
- Hypothesis: H4b
- JVM: `Character/isWhitespace` returns false for U+2007 -- it's a symbol char.
- CLJS: range check `(<= 0x2000 c 0x200A)` includes U+2007 -- it's whitespace.
- Same `.meme` file parses differently on JVM vs CLJS.
- Location: `tokenizer.cljc:41-51`

**F14. `flat/format-clj` missing input validation (silent corruption)**
- Hypothesis: H31b
- `format-clj(nil)` returns `""`. `format-clj("hello")` iterates chars. `format-clj({:a 1})` returns `"[:a 1]"`.
- Location: `flat.cljc:29-36` -- `format-clj` doesn't call `validate-format-input`.

**F15. CLI `--width abc` dumps raw stack trace**
- Hypothesis: H51
- `bb meme format --width abc` shows full babashka.cli coerce exception instead of clean error.
- Location: `cli.clj` -- no catch for coerce failures.

### P3 -- Low

**F16. Stale doc: NBSP treatment contradicts code**
- Hypothesis: H4a
- `doc/design-decisions.md:392-394` says NBSP is part of symbols. Code rejects it (RT6-F8 fix). Doc not updated.

**F17. Bare `\r` in `:ws` metadata leaks through comment extraction**
- Hypothesis: H25
- `str/split-lines` doesn't split on bare `\r`. Only affects legacy Mac line endings.
- Location: `printer.cljc:21`

**F18. `validate-format-input` incomplete type coverage**
- Hypothesis: H31a
- Accepts numbers, keywords, booleans -- these crash downstream. Guards only nil, string, map, set.

**F19. Registry accepts empty-string extension**
- Hypothesis: H56
- `register!` with `{:extension ""}` succeeds. Matches files ending in `.`.

**F20. `format --check /dev/null` misleading message**
- Hypothesis: H48
- Reports "No matching files found" for a file that exists but lacks `.meme` extension.

## Static Analysis Summary

- **clj-kondo**: Zero diagnostics on all `src/meme/` files
- **clojure-lsp**: Clean symbol hierarchies, no anomalies
- **Security**: `requiring-resolve` on EDN-supplied symbols is documented risk (P2-informational); `..` path traversal guard does not block absolute paths/symlinks (P3)

## Priority Fix Recommendations

1. **P0 F1**: Implement `#?@` splice in `read-children` -- check `:meme/splice` metadata and flatten results
2. **P1 F2**: Use a sentinel (e.g., `::no-match`) in reader-conditional eval to distinguish "matched nil" from "no match"
3. **P1 F3/F4**: Reject bare `#::{}` at parse time (Clojure requires a namespace or alias after `#::`)
4. **P2 F5/F6**: Add duplicate detection in map/set construction in `cst_reader.cljc`
5. **P2 F9**: Validate ratio denominator has no sign character
6. **P2 F11**: Validate `::` keyword structure at read time (reject bare `::`, `::/`, trailing `/`)
7. **P2 F14**: Add `validate-format-input` call to `flat/format-clj`
