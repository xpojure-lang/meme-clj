## Executive Summary

The codebase is architecturally sound — the three-pipeline design, stage composition, and pure/eval split are all well-justified. The main findings cluster into four themes:

1. **Two correctness bugs** (P0): invalid octals `08`/`09` parsed silently as decimal; `\null`/`\delete` char literals rejected
2. **Trailing comments lost** in two emission paths (`format-clj` and `rewrite/emit`)  
3. **Significant DRY opportunities** — 3 lang backends with identical `format-meme`, duplicated StringBuilder helpers, 4 sites using `(partition 2 (rc-form ...))` with no shared helper
4. **Testing gaps** — no generative test for `clj→meme` direction, no three-pipeline agreement property, untested macros (`defrule`/`defrule-guard`/`ruleset`), no timing assertions in benchmarks

## Dimension Summaries

### 1. Performance (P) — 85 hypotheses tested

The P1 flat-layout optimization from White Team 1 is shipped. Remaining opportunities:
- **Pre-compute flat widths** on Doc nodes → O(1) `fits?` per group (Wadler-Lindig SOTA)
- **Add `fill` combinator** for flexible argument wrapping (Prettier-style)
- **Guard `extract-comments`** with `(str/includes? ws ";")` — avoids `split-lines` + regex on every form
- **Cache `(count tokens)`** in parser — eliminates repeated O(1) but frequent calls
- **Transient vectors** in `parse-forms-until` for large collections
- **Single-pass `#()` analysis** — merge find/validate/normalize into one tree walk

### 2. DRY (D) — 80 hypotheses tested

The previous White Team's 6 duplicated form walkers remain. New findings:
- **3 identical `format-meme` bodies** across lang backends → extract to `lang/shared.cljc`
- **StringBuilder helpers** duplicated in tokenizer + render → extract to a shared `.cljc` util
- **4-site `rc-pairs` pattern** → introduce `(forms/rc-pairs form)` helper
- **Octal/hex guards** copy-pasted between JVM/CLJS branches in resolve → extract shared predicates

### 3. Architecture (A) — 72 hypotheses tested

The three-pipeline architecture is validated:
- **Classic**: best errors, full feature parity, canonical formatter
- **Rewrite-tree**: explicit rule separation, custom-parser drop-in, guest-language support
- **TRS**: zero-overhead text conversion, ideal for preprocessing/highlighting

Key architectural gap: **No `DocAlign` combinator** — the Doc algebra only supports fixed-indent nesting, not column-relative alignment. This prevents Clojure-style "hanging indent" formatting.

### 4. Testing (T) — 80 hypotheses tested, 48 confirmed gaps

Critical gaps:
- **No generative `clj→meme` roundtrip** — tests only the `meme→forms` direction
- **No three-pipeline agreement property** — only checked for fixed fixtures
- **Zero coverage for JVM macros** (`defrule`, `defrule-guard`, `ruleset`)
- **Benchmark has no timing assertions** — reports numbers but can't catch regressions
- Missing test files for `emit/values.cljc` and `lang/shared.cljc`

### 5. API/UX (U) — 68 hypotheses tested

- **No stdin support** — can't pipe into `bb meme run`
- **No width-aware Clojure output** — `to-clj` is always flat single-line
- **REPL lacks readline/history** — relies on terminal emulator
- **`expand-forms` not in public API** — docstring says one thing, api.md says another
- **All `to-clj` two-arity forms discard opts** — `:read-cond` etc. can't flow through lang dispatch

### 6. CLJS Parity (C) — covered within each dimension

Few true gaps. The CLJS-specific `make-sb` uses JS arrays (correct). `named-chars` missing `\null`/`\delete` affects both platforms. Column tracking UTF-16 vs codepoint issue is shared.

### 7. Error Handling (E) — covered across all dimensions

Generally excellent — all error sites include line/col, secondary locations for unclosed forms. One gap: **`resolve-number` octal error path** where `08` should throw but doesn't.

### 8. Web Research Findings

**Wadler-Lindig SOTA:**
- Pre-compute flat widths → O(1) `fits?` per group
- `fill` combinator for flexible wrapping (Prettier-style)
- `align` combinator for column-relative indentation
- Bernardy's optimal printer (2017) — overkill for code formatting

**Clojure Performance:**
- Transients for hot-loop collection building (tokenizer, parser)
- `case` on keywords (compiles to tableswitch) > protocols > multimethods
- StringBuilder behind `#?(:clj ...)` for string building
- Babashka/SCI: algorithmic complexity matters more than micro-optimization
